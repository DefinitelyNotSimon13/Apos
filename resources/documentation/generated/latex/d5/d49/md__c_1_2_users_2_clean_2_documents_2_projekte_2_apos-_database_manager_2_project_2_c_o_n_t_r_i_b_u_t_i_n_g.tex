If a maintainer asks you to “rebase” your PR, they\textquotesingle{}re saying that a lot of code has changed, and that you need to update your branch, so it\textquotesingle{}s easier to merge.

To learn more about rebasing in Git, there are a lot of \href{https://git-scm.com/book/en/v2/Git-Branching-Rebasing}{\texttt{ good}} \href{https://www.atlassian.com/git/tutorials/merging-vs-rebasing}{\texttt{ resources}} but here\textquotesingle{}s the suggested workflow\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{git\ checkout\ 325-\/add-\/japanese-\/translations}
\DoxyCodeLine{git\ pull\ -\/-\/rebase\ upstream\ main}
\DoxyCodeLine{git\ push\ -\/-\/force-\/with-\/lease\ 325-\/add-\/japanese-\/translations}

\end{DoxyCode}

A PR can only be merged into main by a maintainer if\+:


\begin{DoxyItemize}
\item It is passing CI.
\item It has been approved by at least two maintainers. If it was a maintainer who opened the PR, only one extra approval is needed.
\item It has no requested changes.
\item It is up to date with the current main.
\end{DoxyItemize}

Any maintainer is allowed to merge a PR if these conditions are met.
